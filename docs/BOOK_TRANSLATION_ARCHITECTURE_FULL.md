# Резюме для руководства

## Что предлагается

Создание **промышленной системы перевода книг** на базе существующей инфраструктуры n8n с интеграцией:
- **LightRAG** — система управления знаниями (RAG) для сохранения контекста перевода
- **Ollama** — локальная LLM для перевода и контроля качества

## Ключевые преимущества

| Показатель | Текущее состояние | После внедрения |
|------------|-------------------|-----------------|
| Качество перевода | Зависит от оператора | Консистентное, score ≥8.0 |
| Скорость | ~500 слов/час | ~2000 слов/час (4x) |
| Консистентность терминов | Ручная проверка | Автоматический глоссарий |
| Мониторинг | Отсутствует | Real-time dashboard |
| Отказоустойчивость | Ручное вмешательство | Auto-retry, checkpointing |

## Стоимость владения

| Компонент | Стоимость |
|-----------|-----------|
| LightRAG | Бесплатно (open source) |
| Ollama + модели | Бесплатно (локально) |
| n8n | Уже развёрнут |
| PostgreSQL | Уже развёрнут |
| Grafana/Prometheus | Уже развёрнуты |
| **Итого CAPEX** | **0 ₽** (используем существующее) |
| **OPEX** | **~500 ₽/мес** (electricity) |

## Требуемые ресурсы

- **Время разработки:** 5-7 дней
- **Время тестирования:** 2-3 дня
- **Инфраструктура:** Существующий сервер (Docker)
- **Человеко-часы:** 40-50 часов

# Проблема и решение

## Проблема

### Текущее состояние

```
┌─────────────────────────────────────────────────────────────┐
│                    CURRENT STATE                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Пользователь → Telegram → n8n → Перевод (API) → Результат │
│                           │                                 │
│                           ▼                                 │
│                    ❌ ПРОБЛЕМЫ:                             │
│                    • Нет контекста между переводами         │
│                    • Термины переводятся по-разному         │
│                    • Нет контроля качества                  │
│                    • Нет мониторинга процесса               │
│                    • Ошибки требуют ручного вмешательства   │
│                    • Нет логирования                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### Боли пользователей

1. **Несогласованность терминов**
   - Глава 1: "wizard" → "волшебник"
   - Глава 5: "wizard" → "маг"
   - Глава 10: "wizard" → "чародей"

2. **Потеря контекста**
   - Персонаж меняет имя в середине книги
   - Локации называются по-разному

3. **Отсутствие видимости**
   - "Где мой перевод?"
   - "Сколько ещё ждать?"
   - "Что сломалось?"

4. **Ручная работа**
   - Проверка каждой главы
   - Исправление ошибок вручную
   - Нет автоматического retry

## Решение

```
┌─────────────────────────────────────────────────────────────────┐
│                      PROPOSED STATE                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Пользователь → Telegram/Webhook → n8n Master Workflow         │
│                                    │                            │
│           ┌────────────────────────┼────────────────────┐      │
│           │                        │                    │      │
│           ▼                        ▼                    ▼      │
│    ┌─────────────┐         ┌─────────────┐      ┌──────────┐  │
│    │   Glossary  │         │   Chapter   │      │ Quality  │  │
│    │ Extraction  │         │ Translation │      │  Check   │  │
│    │  (LightRAG) │         │  (LightRAG) │      │ (Ollama) │  │
│    └─────────────┘         └─────────────┘      └──────────┘  │
│           │                        │                    │      │
│           └────────────────────────┼────────────────────┘      │
│                                    │                            │
│           ┌────────────────────────┼────────────────────┐      │
│           │                        │                    │      │
│           ▼                        ▼                    ▼      │
│    ┌─────────────┐         ┌─────────────┐      ┌──────────┐  │
│    │ PostgreSQL  │         │   Grafana   │      │Prometheus│  │
│    │   (Logs)    │         │ (Dashboard) │      │ (Metrics)│  │
│    └─────────────┘         └─────────────┘      └──────────┘  │
│                                                                 │
│  ✅ ПРЕИМУЩЕСТВА:                                               │
│  • Консистентность через глоссарий и RAG                        │
│  • Автоматический контроль качества                             │
│  • Real-time мониторинг                                         │
│  • Auto-retry при ошибках                                       │
│  • Полное логирование                                           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### Решаемые проблемы

| Проблема | Решение | Результат |
|----------|---------|-----------|
| Несогласованность терминов | Glossary Extraction + LightRAG | Единый глоссарий на всю книгу |
| Потеря контекста | RAG context retrieval | Контекст из предыдущих глав |
| Отсутствие видимости | Grafana Dashboard | Real-time прогресс |
| Ручная работа | Automated pipeline + Error Handler | Минимум вмешательства |

# Архитектурное видение

## Принципы проектирования

### 1. Non-breaking Integration
> **Принцип:** Не ломать существующую архитектуру

```
┌─────────────────────────────────────────────────────────┐
│              EXISTING ARCHITECTURE                      │
├─────────────────────────────────────────────────────────┤
│  n8n-docker (34 часа uptime)                            │
│  ├── db (PostgreSQL 16)                                 │
│  ├── n8n (latest)                                       │
│  ├── pgadmin4                                           │
│  ├── caddy (bigalexn8n.ru)                              │
│  └── monitoring stack                                   │
│      ├── prometheus                                     │
│      ├── grafana                                        │
│      ├── node-exporter                                  │
│      └── postgres-exporter                              │
│                                                         │
│  lightrag (16 минут uptime)                             │
│  ├── ollama (qwen2.5:32b, llama3.2:3b)                  │
│  └── lightrag                                           │
└─────────────────────────────────────────────────────────┘

✅ НОВАЯ СИСТЕМА:
  • Использует СУЩЕСТВУЮЩИЕ контейнеры
  • Добавляет НОВЫЕ таблицы в БД (без изменений схемы)
  • Добавляет НОВЫЕ workflows (не модифицирует старые)
  • Добавляет НОВЫЙ dashboard в Grafana
```

### 3. Fault Tolerance by Design

```
┌─────────────────────────────────────────────────────────────┐
│                  RESILIENCE PATTERNS                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  CHECKPOINTING                                       │  │
│  │  После каждой главы → сохранение в БД                │  │
│  │  При сбое → возобновление с последней сохранённой   │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                             │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  RETRY WITH EXPONENTIAL BACKOFF                      │  │
│  │  Попытка 1: wait 10s                                 │  │
│  │  Попытка 2: wait 30s                                 │  │
│  │  Попытка 3: wait 90s                                 │  │
│  │  Попытка 4: FAIL (уведомление)                       │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                             │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  CIRCUIT BREAKER                                     │  │
│  │  Ollama: 5 ошибок за 2 мин → circuit OPEN (60 сек)   │  │
│  │  LightRAG: 3 ошибки за 1 мин → circuit OPEN (30 сек) │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                             │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  DEAD LETTER QUEUE                                   │  │
│  │ _failed_chapters таблица для ручного разбора         │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

# Обоснование технологических решений

## Почему LightRAG?

### Проблема контекста

При переводе книг критически важен **контекст**:
- Термины должны переводиться единообразно
- Имена собственные должны сохраняться
- Стиль должен быть консистентным

### Решение LightRAG

```
┌─────────────────────────────────────────────────────────────────┐
│                  WHY LIGHTRAG?                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  RAG = Retrieval-Augmented Generation                          │
│                                                                 │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │  БЕЗ RAG:                                                │  │
│  │  ────────────                                            │  │
│  │  Глава 1: "The wizard cast a spell"                      │  │
│  │  → "Волшебник произнёс заклинание"                       │  │
│  │                                                          │  │
│  │  Глава 10: "The wizard cast a spell"                     │  │
│  │  → "Маг произнёс заклинание"  ❌ НЕСОГЛАСОВАННО          │  │
│  └──────────────────────────────────────────────────────────┘  │
│                                                                 │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │  С RAG:                                                  │  │
│  │  ────────                                                │  │
│  │  1. Извлекаем глоссарий из всей книги                    │  │
│  │  2. При переводе главы 10:                                │  │
│  │     • Query: "wizard translation"                        │  │
│  │     • Retrieve: глава 1 → "волшебник"                    │  │
│  │     • Context: "wizard = волшебник"                      │  │
│  │  3. Перевод с контекстом:                                 │  │
│  │     → "Волшебник произнёс заклинание" ✅ КОНСИСТЕНТНО    │  │
│  └──────────────────────────────────────────────────────────┘  │
│                                                                 │
│  ПРЕИМУЩЕСТВА LIGHTRAG:                                        │
│  ✓ Already deployed (host networking, port 9621)               │
│  ✓ Интеграция с Ollama (embedding: nomic-embed-text)           │
│  ✓ Автоматическое управление контекстом                        │
│  ✓ Chunking + embeddings из коробки                            │
│  ✓ API для query и document upload                             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

| **LightRAG** | ✅ Уже развёрнут, host networking, simple API |

## Матрица рисков

```
┌─────────────────────────────────────────────────────────────────┐
│                      RISK MATRIX                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  IMPACT                                                         │
│    High │  ┌─────────────┐  ┌─────────────┐                    │
│         │  │ GPU Failure │  │ Data Loss   │                    │
│         │  │ (Mitigation)│  │ (Backup)    │                    │
│         │  └─────────────┘  └─────────────┘                    │
│         │  ┌─────────────┐  ┌─────────────┐                    │
│  Medium │  │ Ollama      │  │ API Rate    │                    │
│         │  │ Slowdown    │  │ Limiting    │                    │
│         │  └─────────────┘  └─────────────┘                    │
│         │  ┌─────────────┐                                     │
│    Low  │  │ UI Bug      │                                     │
│         │  └─────────────┘                                     │
│         └──────────────────────────────────────────             │
│              Low        Medium        High      PROBABILITY     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## Детальная оценка рисков

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| **GPU failure** | Low | High | • Fallback на CPU (медленнее)<br>• Retry queue<br>• Уведомление админу |
| **Data loss** | Low | High | • PostgreSQL backup (ежедневно)<br>• Checkpointing после каждой главы<br>• WAL logging |
| **Ollama slowdown** | Medium | Medium | • Circuit breaker<br>• Queue management<br>• Load monitoring |
| **API rate limiting** | Medium | Medium | • Exponential backoff<br>• Request queuing<br>• Multiple API keys (future) |
| **Quality degradation** | Low | Medium | • Quality threshold<br>• Manual review queue<br>• A/B testing |
| **Memory overflow** | Low | Medium | • Batch size limits<br>• Memory monitoring<br>• Auto-restart |
| **Network issues** | Medium | Low | • Retry logic<br>• Local services (no external deps)<br>• Health checks |

    metadata JSONB DEFAULT '{}'::jsonb,
    requested_by VARCHAR(200),
    telegram_chat_id BIGINT
);

-- Chapters table
CREATE TABLE IF NOT EXISTS book_chapters (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

    translated_text TEXT,

    created_at TIMESTAMPTZ DEFAULT NOW(),

    completed_at TIMESTAMPTZ,
    error_message TEXT,
    retry_count INTEGER DEFAULT 0,
    context_chunks JSONB DEFAULT '[]'::jsonb,
    glossary_terms JSONB DEFAULT '[]'::jsonb,
    metadata JSONB DEFAULT '{}'::jsonb
);

-- Glossary table
CREATE TABLE IF NOT EXISTS book_glossary (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    job_id UUID REFERENCES book_translation_jobs(id) ON DELETE CASCADE,
    term VARCHAR(200) NOT NULL,
    translation VARCHAR(200) NOT NULL,
    part_of_speech VARCHAR(50),
    context TEXT,
    frequency INTEGER DEFAULT 1,
    confidence_score DECIMAL(5,2),
    created_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(job_id, term)
);

-- Statistics table
CREATE TABLE IF NOT EXISTS book_translation_stats (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    job_id UUID REFERENCES book_translation_jobs(id) ON DELETE CASCADE,
