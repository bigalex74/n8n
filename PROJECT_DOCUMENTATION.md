# 📗 Документация проекта: Novel Translation Pipeline (v3)

## 🏗 Жизненный цикл перевода книги

### 1. Подготовка (Setup)
- **Вход:** `Start` или `Ручной выбор файлов`.
- **Анализ:** `Предварительный анализ...` разбивает текст на главы и извлекает глоссарий. Данные сохраняются в таблицы `document_chapters`, `document_arcs` и `document_glossary`.

### 2. Главный Цикл (Master Loop)
- **Управление:** `Translate Chunk` — центральный воркфлоу. Он итерирует по всем чанкам книги.
- **Действия внутри цикла:**
    1. Вызов `[Перевод] Перевод чанка` для трансформации текста.
    2. Вызов `[Перевод] Глава` / `[Перевод] Арка` для детекции границ и создания суммаризаций (context updates).

### 3. Финализация (Finalize)
- **Сборка:** `Finish` объединяет переведенные фрагменты из `document_chunks` в единый документ и выводит статистику затрат.

## 🗄 Структура Базы Данных (Postgres)
- `document_jobs`: Реестр книг и глобальный статус.
- `document_chunks`: Атомарные части текста (оригинал + перевод).
- `document_glossary`: База имен и терминов.
- `telegram_send_message`: Очередь для отправки уведомлений через оркестратор `Send Message`.

## 📜 Правила и Особенности
- **Translate Chunk** — критический узел. Любые изменения в логике перебора чанков делаются здесь.
- **[Перевод]...** — специализированные воркфлоу-функции. Они должны принимать `job_id` и возвращать статус выполнения.
- **Idempotency:** Все уведомления в Telegram проходят через `Send Message` для исключения спама.
