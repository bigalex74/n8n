# 📗 Документация проекта: Novel Translation Pipeline

Система автоматизированного перевода и анализа корейских новелл на базе n8n.

## 🏗 Архитектура системы
Проект разделен на 3 логических уровня:
1. **Триггеры:** Telegram-бот и прямые вставки в базу данных PostgreSQL.
2. **Оркестратор:** Главный воркфлоу `Send Message`, управляющий жизненным циклом задачи.
3. **Сервисы (Sub-workflows):** Модули для уведомлений, получения контекста и обработки чанков.

## 🗄 База данных (PostgreSQL)
Используются две БД:
- `n8n_database`: Системные данные n8n.
- `postgres`: Основная БД приложения.
  - `document_jobs`: Статусы заданий и биллинг.
  - `document_chunks`: Текст для перевода.
  - `document_glossary`: Глоссарий (извлекается LLM).
  - `v_novel_dashboard`: Представление для аналитики.

## 🚀 Основные воркфлоу
- **Send Message:** Сердце системы. Маршрутизирует задачи (Create -> Start -> Process -> Finish).
- **System - Proxy Check:** Мониторинг доступности прокси 10808.
- **System - Novel Pipeline Test:** Автоматизированный тест всей цепочки.
- **System - Stats Dashboard:** Глобальная аналитика по всем новеллам.

## 🛠 Технические требования
- **Proxy:** HTTP/HTTPS прокси на `127.0.0.1:10808` (обязательно для Telegram и AI).
- **LLM:** Используются модели DeepSeek (через Polza API) и Qwen.
- **Git Sync:** Автоматический бэкап каждые 4 часа в репозиторий `bigalex74/n8n`.

## 📜 Правила разработки
1. Всегда проверять логи Docker после изменений.
2. Использовать `Edit Message` вместо дублирования сообщений в Telegram.
3. Коммитить изменения только через ветки `bigalexn8n-X`.
